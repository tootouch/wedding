## 공통 정보 구조

거의 모든 서비스가 아래 순서를 기본값으로 쓴다.

1. 메인 커버: 사진 또는 일러스트, 신랑신부 이름, 날짜
2. 초대 문구: 짧은 감성 문장 + 정중한 본문
3. 예식 정보: 날짜, 시간, 장소, D-Day
4. 혼주/연락처: 신랑신부와 양가 부모님 연락
5. 갤러리: 웨딩 사진 중심, 10~30장 사이가 일반적
6. 오시는 길: 지도앱 링크, 교통편, 주차, 대절버스
7. 마음 전하실 곳: 접힘/복사/송금 기능
8. RSVP: 참석 여부, 인원, 식사 여부
9. 방명록/축하 메시지
10. 공유: 카카오톡, 문자, URL, QR

## 디자인 패턴

- 전통형: 아이보리, 크림, 베이지, 골드, 세리프 서체, 중앙 정렬, 종이 질감.
- 감성형: 풀스크린 사진, 오프닝 모션, 계절 효과, 얇은 선, 작은 영문 키커.
- 매거진형: 큰 사진과 짧은 문장을 번갈아 배치, 섹션 간 호흡이 길다.
- 미니멀형: 장식 최소화, 이름/날짜/장소를 아주 또렷하게 보여준다.
- 프리미엄형: 네이비/골드, 플라워/리프, 후가공 느낌의 경계선과 여백.

## UX 패턴

- 빠른 메뉴는 상단 또는 하단에 고정하면 실사용성이 좋다.
- 지도, 계좌, 전화, 문자, 공유는 버튼으로 즉시 실행되어야 한다.
- 계좌 정보는 처음부터 노출하기보다 접힘/복사 구조가 부담이 적다.
- RSVP는 너무 늦게 두면 제출률이 낮아질 수 있어 중간 또는 진입 팝업이 좋다.
- 부모님용/친구용처럼 대상별 공유 문구가 있으면 실제 전달 상황에 유리하다.
- 하객은 앱 설치 없이 브라우저에서 바로 볼 수 있어야 한다.

## 현재 1차 시안과 비교

이미 구현됨:

- 풀스크린 커버
- 빠른 메뉴
- 초대 문구
- 예식 정보와 카운트다운
- 연락처
- 갤러리
- 지도 링크
- 계좌 복사
- RSVP
- 공유

부족한 점:

- 실제 웨딩 사진 기반 갤러리 다양성이 부족하다.
- 계좌 섹션이 처음부터 열려 있어 다소 직접적이다.
- 방명록이 없다.
- RSVP가 로컬 저장만 하고 관리 화면이 없다.
- 카카오톡 공유 SDK/OG 이미지 최적화가 아직 없다.
- 테마 전환 구조가 없다.
- 부모님용/친구용 문구 분리가 없다.

## 다음 개선안

1. 계좌 영역을 접힘 UI로 바꾸기
2. 방명록 섹션 추가
3. RSVP를 “참석/불참 + 식사 + 동행인”으로 확장
4. 실제 사진 5~8장 넣을 수 있는 갤러리 구조로 변경
5. 테마 CSS 변수 추가: 클래식, 매거진, 모던, 필름
6. 카카오 공유용 메타 태그와 썸네일 이미지 정리
7. 부모님용/친구용 공유 문구 프리셋 추가
8. 예식 후 감사 인사 카드 섹션으로 전환 가능한 상태 추가

## 이번 프로젝트의 추천 방향

기본 톤은 “데어무드식 감성 디테일 + 바른손/보자기식 정보 안정성”이 좋다. 즉, 첫 화면은 사진과 여백으로 고급스럽게 만들고, 아래 정보 영역은 어른 하객도 바로 이해할 수 있게 또렷하게 정리한다.

복제하지 않을 것:

- 특정 서비스의 템플릿 레이아웃, 장식, 문구를 그대로 사용하지 않는다.
- 디즈니/캐릭터/브랜드 협업 이미지는 쓰지 않는다.
- 리뷰 문구나 샘플 커플 정보를 복사하지 않는다.

우리가 만들 것:

- 실제 두 사람의 사진과 예식 정보를 중심으로 한 오리지널 모바일 청첩장.
- 섹션 순서와 기능을 쉽게 바꿀 수 있는 작은 정적 앱.
- 카카오톡으로 보냈을 때 예쁘게 보이는 모바일 우선 페이지.
